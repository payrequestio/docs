# Payment Pages 💰

PayRequest Payment Pages are powerful, customizable payment collection tools that enable you to receive payments for any purpose without requiring customers to browse your shop. Perfect for invoicing, donations, service fees, deposits, or any custom payment scenario.

<Info>
**Flexible Payment Collection**: Payment Pages work independently of your product catalog, allowing customers to pay custom amounts or predefined fees with a clean, professional interface.
</Info>

## 🛠️ Setting Up Your Payment Page

### Configuration Process

<Steps>
  <Step title="Access Payment Page Settings">
    Navigate to **Payment Pages** in your PayRequest dashboard or go directly to `/paymentpage`
  </Step>

  <Step title="Configure Basic Settings">
    **Essential Configuration:**
    - **Custom Handle**: Create a memorable URL (e.g., `payrequest.me/yourhandle`)
    - **Page Title**: Professional heading (e.g., "Pay YourBusiness")
    - **Description**: Clear payment instructions
    - **Amount Settings**: Default, minimum, and maximum amounts
  </Step>

  <Step title="Customize Payment Options">
    **Advanced Settings:**
    - Enable/disable custom amount entry
    - Require customer descriptions
    - Set success messages
    - Configure redirect URLs
    - Enable dynamic field controls
  </Step>

  <Step title="Test and Activate">
    **Go Live Process:**
    - Test with small amounts
    - Verify payment processing
    - Check email notifications
    - Confirm mobile experience
    - Activate for public use
  </Step>
</Steps>

### Configuration Options

<Expandable title="⚙️ Basic Settings">
**Core Configuration:**

**Custom Handle:**
- Create memorable URLs (letters, numbers, hyphens only)
- Example: `payrequest.me/consultation` or `payrequest.me/donate`
- Auto-converts to lowercase
- Must be unique across PayRequest
- Falls back to business name if not set

**Page Content:**
- **Title**: Main heading (e.g., "Pay for Services")
- **Description**: Instructions or context for payment
- **Success Message**: Custom thank-you message after payment
- All text supports your brand voice and tone
</Expandable>

<Expandable title="💰 Amount Configuration">
**Payment Amount Controls:**

**Default Amount:**
- Pre-fill amount field with suggested value
- Customers can modify if custom amounts enabled
- Useful for standard fees or donations
- Leave blank for completely open amounts

**Amount Limits:**
- **Minimum**: Prevent micro-payments (default €0.01)
- **Maximum**: Set upper payment limits
- **Custom Amounts**: Allow/disable customer amount entry
- Real-time validation and error messages

**Amount Display:**
- Euro (EUR) currency formatting
- Automatic decimal handling
- Clear visual feedback
- Mobile-friendly input
</Expandable>

<Expandable title="📋 Form Behavior">
**Customer Experience Controls:**

**Information Requirements:**
- **Name**: Optional or required customer name
- **Email**: For receipt delivery
- **Description**: Allow customers to add payment notes
- **Readonly Fields**: Lock specific form elements

**Advanced Options:**
- **Require Description**: Force customers to add payment context
- **Dynamic Field Control**: Enable/disable field modifications
- **Guest Payments**: Allow payments without account creation
- **Auto-fill Integration**: Support logged-in user data
</Expandable>
