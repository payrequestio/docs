# Pre-filled Checkout URLs

Send your customers direct links to purchase products with their information already filled in. Pre-filled checkout URLs save time for your customers and increase conversion rates by removing friction from the buying process.

## How It Works

Instead of sending customers to your shop where they have to browse and add products to cart, you can create direct checkout links that:

- Pre-add specific products to their cart
- Pre-fill their name, email, and address information
- Include tracking information or special requirements
- Take them straight to the payment step

## Basic URL Structure

Pre-filled checkout URLs follow this format:

```
https://yourshop.com/checkout?parameter=value&parameter2=value2
```

Or if you're using the main PayRequest domain:
```
https://payrequest.app/shop/yourshopname/checkout?parameter=value&parameter2=value2
```

<Info>
Replace `yourshop.com` with your actual shop domain and `yourshopname` with your shop's name in PayRequest.
</Info>

## Customer Information Parameters

Pre-fill your customer's information to save them time:

| Parameter | Description | Example |
|-----------|-------------|---------|
| `name` | Customer's full name | `name=John%20Doe` |
| `email` | Customer's email address | `email=john@example.com` |
| `address` | Customer's street address | `address=123%20Main%20St` |
| `city` | Customer's city | `city=New%20York` |
| `postal` | Customer's postal/zip code | `postal=10001` |
| `country` | Customer's country (2-letter code) | `country=US` |

<Warning>
Remember to URL-encode special characters like spaces (`%20`) and symbols.
</Warning>

## Product Parameters

Add specific products to the customer's cart:

| Parameter | Description | Example |
|-----------|-------------|---------|
| `product` | Product ID to add to cart | `product=123` |
| `quantity` | Quantity of the product | `quantity=2` |

To find your Product ID, go to your PayRequest dashboard and view the product details. The ID is shown in the URL or product settings.

## Custom Fields

Custom fields let you collect additional information or include tracking data. PayRequest supports three different formats:

<Tabs>
  <Tab title="Standard Format">
    Use `customfield_` prefix for visible fields customers can see:
    
    ```
    customfield_size=Large
    customfield_color=Red
    customfield_notes=Special%20requirements
    ```
    
    These fields appear on the checkout form for customers to see and modify if needed.
  </Tab>
  
  <Tab title="Product-Specific Format">
    Use `product_customfield_` prefix for tracking and internal use:
    
    ```
    product_customfield_productcode=SKU001
    product_customfield_campaign=summer2024
    product_customfield_referrer=partner123
    ```
    
    These fields are typically hidden from customers and used for business tracking.
  </Tab>
  
  <Tab title="Short Format">
    Use `cf_` prefix as a shorter alternative:
    
    ```
    cf_servers=2
    cf_domain=example.com
    cf_priority=high
    ```
    
    This is a compact way to include custom field data.
  </Tab>
</Tabs>

## Practical Examples

### 1. Simple Product Link
Send a customer directly to buy a specific product:

```
https://yourshop.com/checkout?product=123&quantity=1&name=Jane%20Smith&email=jane@example.com
```

This link adds Product #123 to the cart and pre-fills the customer's name and email.

### 2. Complete Customer Information
Include full customer details for faster checkout:

```
https://yourshop.com/checkout?product=456&name=John%20Doe&email=john@example.com&address=123%20Main%20St&city=New%20York&postal=10001&country=US
```

### 3. With Product Options
Include product variations or options:

```
https://yourshop.com/checkout?product=789&quantity=2&customfield_size=Large&customfield_color=Blue&name=Customer%20Name&email=customer@example.com
```

### 4. Campaign Tracking
Track where customers came from:

```
https://yourshop.com/checkout?product=101&product_customfield_campaign=email_newsletter&product_customfield_source=mailchimp&name=Newsletter%20Subscriber&email=subscriber@example.com
```

### 5. Quote Follow-up
Send customers a direct link after providing a quote:

```
https://yourshop.com/checkout?product=555&quantity=5&customfield_quote_id=Q12345&product_customfield_sales_rep=john&name=Business%20Owner&email=owner@company.com
```

## URL Encoding Guide

Always encode special characters in URL parameters:

<CardGroup cols={2}>
  <Card title="Common Characters" icon="code">
    - **Space**: `%20` or `+`
    - **@**: `%40` 
    - **&**: `%26`
    - **=**: `%3D`
    - **#**: `%23`
  </Card>
  <Card title="Examples" icon="link">
    - `John Doe` → `John%20Doe`
    - `user@email.com` → `user%40email.com`
    - `Special & chars` → `Special%20%26%20chars`
  </Card>
</CardGroup>

<Tip>
Most programming languages have built-in URL encoding functions. In JavaScript, use `encodeURIComponent()`. In PHP, use `urlencode()`.
</Tip>

## Validation and Error Handling

PayRequest validates all URL parameters to ensure a smooth checkout experience:

### What Gets Validated

<Tabs>
  <Tab title="Product Validation">
    - Product must exist in your shop
    - Product must be active and available
    - Quantity must be a positive number
  </Tab>
  
  <Tab title="Customer Data">
    - Email addresses must be valid format
    - Country codes must be valid 2-letter codes
    - Postal codes are validated by country
  </Tab>
  
  <Tab title="Custom Fields">
    - Fields must match your product's configuration
    - Number fields must contain valid numbers
    - URL fields must be valid URLs
  </Tab>
</Tabs>

### Error Behavior

If there are issues with the URL parameters:

- **Invalid products**: Ignored, checkout continues without them
- **Invalid custom fields**: Filtered out, valid fields remain
- **Malformed URLs**: Gracefully fall back to normal checkout

<Note>
All validation errors are logged for your review, but customers see a clean checkout experience.
</Note>

## Security and Best Practices

<Warning>
**Never include sensitive information** like passwords, credit card numbers, or private data in checkout URLs. URLs can be logged and cached by browsers and servers.
</Warning>

### Best Practices

<Steps>
  <Step title="Use HTTPS">
    Always use secure HTTPS URLs for checkout links
  </Step>
  <Step title="Validate on Backend">
    Always verify the final order details on your backend systems
  </Step>
  <Step title="Monitor Usage">
    Review checkout URL analytics to optimize conversion
  </Step>
  <Step title="Keep URLs Clean">
    Only include necessary parameters to keep URLs manageable
  </Step>
</Steps>

### Security Features

- **Shop Isolation**: Products are validated to belong to your shop only
- **Field Validation**: Custom fields must match your product configuration
- **Audit Logging**: All URL parameter usage is logged for review
- **Session Protection**: Existing customer sessions are preserved

```bash Test URL

https://yourshop.com/checkout?product=123&name=Test%20Customer&email=test@example.com
```

```javascript Generate URLs in JavaScript
function generateCheckoutUrl(domain, productId, customerData, customFields = {}) {
  const params = new URLSearchParams();
  
  // Add product
  params.append('product', productId);
  
  // Add customer data
  if (customerData.name) params.append('name', customerData.name);
  if (customerData.email) params.append('email', customerData.email);
  if (customerData.address) params.append('address', customerData.address);
  
  // Add custom fields
  Object.entries(customFields).forEach(([key, value]) => {
    params.append(`customfield_${key}`, value);
  });
  
  return `https://${domain}/checkout?${params.toString()}`;
}

// Example usage
const url = generateCheckoutUrl('yourshop.com', 123, {
  name: 'John Doe',
  email: 'john@example.com'
}, {
  size: 'Large',
  color: 'Blue'
});
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Product Not Added to Cart" icon="shopping-cart">
    **Possible causes:**
    - Product ID doesn't exist
    - Product is inactive or unavailable
    - Product belongs to a different shop
    
    **Solution:** Verify the product ID in your PayRequest dashboard
  </Accordion>

  <Accordion title="Customer Information Not Pre-filled" icon="user">
    **Possible causes:**
    - Parameters not URL-encoded properly
    - Typos in parameter names
    - Special characters not encoded
    
    **Solution:** Check parameter spelling and use URL encoding
  </Accordion>

  <Accordion title="Custom Fields Not Showing" icon="tags">
    **Possible causes:**
    - Custom field doesn't exist for the product
    - Wrong prefix format used
    - Field type mismatch
    
    **Solution:** Verify custom field configuration in product settings
  </Accordion>

  <Accordion title="URL Too Long" icon="link">
    **Possible causes:**
    - Too many parameters
    - Long custom field values
    - Complex product configurations
    
    **Solution:** Use shorter field names or break into multiple steps
  </Accordion>
</AccordionGroup>
