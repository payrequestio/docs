## Payment Method Configuration

### Automatic Selection

By default, PayRequest automatically shows the most relevant payment methods based on:

- **Customer Location**: Detected from IP address or billing country
- **Currency**: Payment methods available for the transaction currency
- **Amount**: Some methods have minimum/maximum limits
- **Device Type**: Mobile-optimized methods for mobile devices

### Manual Configuration

You can customize which payment methods to show:

<Tabs>
  <Tab title="Dashboard">
    1. Go to **Settings > Payment Methods**
    2. Toggle payment methods on/off
    3. Set country/currency restrictions
    4. Configure method-specific settings
  </Tab>

  <Tab title="API">
    ```json
    {
      "amount": 2500,
      "currency": "USD",
      "payment_methods": {
        "card": true,
        "apple_pay": true,
        "google_pay": true,
        "paypal": false,
        "bank_transfer": false
      }
    }
    ```
  </Tab>
</Tabs>

## Optimizing Payment Methods

### Best Practices

<CardGroup cols={2}>
  <Card title="Localization" icon="globe">
    - Show local payment methods first
    - Use local currency when possible
    - Translate payment method names
    - Display local bank logos
  </Card>
  <Card title="Mobile Optimization" icon="mobile">
    - Prioritize digital wallets on mobile
    - Enable card scanning with camera
    - Use biometric authentication
    - Optimize for thumb navigation
  </Card>
</CardGroup>

### A/B Testing

Test different payment method configurations:

1. **Method Selection**: Test showing more vs fewer options
2. **Order**: Test different payment method ordering
3. **Styling**: Test visual presentation of methods
4. **Defaults**: Test pre-selecting preferred methods
